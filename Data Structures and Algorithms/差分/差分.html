核心思想（一句废话没有）

生活类比：假设你是一辆公交车的售票员，你不需要在每一站都去数车上一共有多少人。你只需要记下“这站上来了几个人（加法），下去了几个人（减法）”。

  • 暴力法：要给区间 $[L, R]$ 内的所有数字加上 $V$，写个 `for` 循环挨个加。如果有 $M$ 次区间修改，时间复杂度是 $O(N \times M)$，必吃 TLE（Time Limit Exceeded）。
  • 差分法：构造一个“变化量”数组。在 $L$ 处放一个 $+V$ 的标记（代表从这开始加上 $V$），在 $R+1$ 处放一个 $-V$ 的标记（代表加 $V$ 的效果到此结束）。最后从头到尾扫一遍（求前缀和），就能还原出修改后的原数组。

算法本质：区间操作转化为单点操作。将 $O(N)$ 的区间修改降维成 $O(1)$ 的端点修改，是前缀和的逆运算。

XCPC 竞赛中的应用场景

在各类算法竞赛中，差分常常是处理“频繁修改，一次查询”（离线操作）的核心利器。常见套路包括：

  • 二维/多维差分：结合容斥原理，将二维矩阵的区域加法转换为 4 个顶点的单点修改，常用于处理二维图纸上的涂色、覆盖问题。
  • 树上差分：将数组上的差分推广到树的路径上，结合 LCA（最近公共祖先），可以 $O(1)$ 修改树上两点之间路径的权值，最后通过一遍 DFS 回溯还原整棵树的权值。
  • 多次差分（二阶差分）：如果题目要求在区间内加上一个“等差数列”（如依次加 1, 2, 3...），对一阶差分数组再做一次差分，就能把等差数列的修改转化为常数次单点修改。
  • 扫描线与区间重叠：给定大量区间，求重叠次数最多的部分。只需在起点 $+1$，终点后一位 $-1$，求前缀和的最大值即可。

XCPC 必刷经典题单

以下题目涵盖了差分在竞赛中的主流变形考察点，建议按顺序推进：

  • Level 1: 基础测试
    洛谷 P2367 - 语文成绩 1D差分模板
    评语：纯粹的区间加法模板题，用来熟练掌握 D[L] += V; D[R+1] -= V; 的端点修改和最终的前缀和还原。
  • Level 2: 维度跃升
    洛谷 P3397 - 地毯 2D差分
    评语：二维差分入门。核心是理解二维修改公式：D[x1][y1] += 1; D[x2+1][y1] -= 1; D[x1][y2+1] -= 1; D[x2+1][y2+1] += 1;。
  • Level 3: 贪心 + 排序 + 差分
    Codeforces 276C - Little Girl and Maximum Sum 差分求覆盖频次
    评语：非常经典的CF思维题。通过差分求出每个位置被查询的“频次”，然后把原数组中最大的数放在频次最高的位置上，贪心求最大总和。
  • Level 4: 树上路径修改
    洛谷 P3128 - [USACO15DEC] Max Flow P 树上差分 + LCA
    评语：树上点权差分的必刷题。路径 $u$ 到 $v$ 加 1，操作变为 D[u]++, D[v]++, D[lca(u,v)]--, D[parent(lca)]--。
  • Level 5: 进阶数学规律
    洛谷 P4231 - 三步必杀 二阶差分
    评语：处理区间加上等差数列的黑科技。差分数组的差分，做两次前缀和还原，是对数学功底的极好试金石。

模板代码（C++ 版本）

和前缀和一样，强烈推荐使用 1-based 索引（下标从 1 开始）。并且差分数组必须比原数组多开 2 个位置，以防 $R+1$ 越界。

#include <iostream>
#include <vector>

using namespace std;
typedef long long ll; 

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n = 5; // 原数组长度
    // 假设初始数组全为 0。如果初始数组有值，可以直接视作进行了 n 次 [i, i] 的单点区间加法。
    vector<ll> a(n + 2, 0); 
    vector<ll> diff(n + 2, 0); // 差分数组，大小必须开到 n+2
    
    int q = 2; // 假设有 q 次区间修改操作
    // 操作 1: 区间 [2, 4] 加上 10
    int L1 = 2, R1 = 4; ll V1 = 10;
    diff[L1] += V1;
    diff[R1 + 1] -= V1; // R+1 的位置减去 V

    // 操作 2: 区间 [3, 5] 加上 5
    int L2 = 3, R2 = 5; ll V2 = 5;
    diff[L2] += V2;
    diff[R2 + 1] -= V2;
    
    // 最终：通过对差分数组求前缀和，还原出最终的原数组 a
    for (int i = 1; i <= n; i++) {
        a[i] = a[i - 1] + diff[i]; 
        cout << a[i] << " ";
    }
    // 输出应为: 0 10 15 15 5
    
    return 0;
}

赛场避坑指南 (WA / RE 警告)

1. $R+1$ 越界导致段错误 (RE)

差分修改时必写 D[R+1] -= V，如果 $R = N$，那么 $R+1 = N+1$。如果你的数组严格只开了 $N$ 大小，这里就会数组越界引发 RE 或者覆盖其它变量。习惯性把数组大小开到 $N+5$。

2. 离线与在线的混淆

差分数组只适用于离线操作（即：先把所有的“区间修改”全做完，最后只做一次“查询”求前缀和）。如果题目要求边修改边查询（例如：改一下区间，问一下单点，再改一下区间），差分数组会退化回 $O(N)$，此时必须上树状数组 (BIT) 或 线段树。

3. 忘记还原数组 (WA)

差分数组 D 记录的只是变化量！输出最终结果时，务必记得跑一遍 for(i=1~N) a[i] = a[i-1] + d[i];，不要直接把 D 数组当答案输出了。

复杂度分析

  • 时间复杂度：单次区间修改 $O(1)$。$M$ 次修改后，最后求一次前缀和还原数组 $O(N)$。总耗时 $O(N + M)$。
  • 空间复杂度：$O(N)$，需要开一个与原数组同等规模的差分数组。
