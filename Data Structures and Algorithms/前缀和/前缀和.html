核心思想

生活类比：假设你在记账，想知道“本月 5 号到 15 号一共花了多少钱”。

  • 普通人的做法（暴力法）：把 5 号到 15 号的账单翻出来，一笔一笔相加。每次问你，你都要重加一遍。
  • XuRia(Goat)的做法（前缀和）：每天记账时，顺手记下“从 1 号到今天总共花了多少钱”。查询时只需用 “15 号总额” 减去 “4 号总额”。

算法本质：空间换时间。预处理时间复杂度 O(N)，查询时间复杂度 O(1)。

算法竞赛中的应用场景

在算法竞赛（如 ACM/ICPC、蓝桥杯、LeetCode 周赛）中，前缀和极少作为单独的考点，而是作为优化的底层基础设施。常见应用场景包括：

  • 频繁的区间求和查询：如果题目要求对同一个数组进行多次区间查询，直接遍历一定会超时（TLE），前缀和是此类操作的标配。
  • 配合哈希表查找特定子数组：例如经典问题“求和为 K 的连续子数组个数”。通过将区间和公式变形为 prefix[R] - K = prefix[L-1]，配合哈希表记录扫过的前缀和出现的次数，可以将 O(N^2) 的算法降维到 O(N)。
  • 差分数组的逆运算：在对区间进行频繁的加减操作后，对差分数组求一次前缀和，即可在 O(N) 时间内还原出最终的数组状态。

经典竞赛例题

  • 入门模板：洛谷 P8218 - 【深进1.例1】求区间和 （适合用来测试自己写的模板是否有 Bug）
  • 基础封装：LeetCode 303 - 区域和检索 (数组不可变) （考察面向对象编程中的前缀和封装）
  • 进阶应用：LeetCode 560 - 和为 K 的子数组 （高频面试题：前缀和 + 哈希表优化的经典结合）

模板代码（C++ 版本）

推荐使用 1-based 索引，这是写前缀和最优雅且不易出错的方式。

#include <iostream>
#include <vector>

using namespace std;

int main() {
    int n = 8; 
    vector<int> nums = {0, 3, 1, 4, 1, 5, 9, 2, 6}; // 0位占位
    
    // 1. 预处理前缀和数组
    vector<long long> prefix(n + 1, 0); 
    for (int i = 1; i <= n; i++) {
        prefix[i] = prefix[i - 1] + nums[i]; 
    }
    
    // 2. 查询区间 [L, R]
    int L = 2, R = 5;
    long long sum = prefix[R] - prefix[L - 1]; 
    
    cout << "Sum: " << sum << endl;
    return 0;
}

易错点避坑指南

1. 数组越界

如果 L=1，公式会访问 prefix[0]。这就是为什么要多开一位空间！ 让 prefix[0] = 0，代码就不需要特判 L 是否为 1，逻辑可以直接自洽。

2. 数据溢出

在算法竞赛中，多个 int 相加极易超过整型最大值（约 2x10^9）。前缀和数组请务必习惯性使用 long long 进行声明。

复杂度分析

  • 时间复杂度：预处理阶段遍历原数组，时间复杂度为 O(N)。单次查询仅需一次减法，时间复杂度为 O(1)。
  • 空间复杂度：需要开辟大小为 N+1 的数组存储前缀和，空间复杂度为 O(N)。
