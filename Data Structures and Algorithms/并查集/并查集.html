核心思想

生活类比：想象江湖上有多个帮派。起初每个人都是自成一派。帮派之间会发生合并（Union），合并时需要确定谁是帮主。如果你想知道两个人是否属于同一帮派（Find），只需看他们的“最高帮主”是否为同一人即可。

  • 查询 (Find)：找寻一个元素所属集合的代表元（帮主）。通过“路径压缩”，我们可以让所有成员直接向帮主汇报，极大提升速度。
  • 合并 (Union)：将两个不相交的集合合并为一个。通过“按秩合并”，我们可以让小帮派并入大帮派，防止帮派层级过深。

算法本质：维护集合的隶属关系。它能以近乎 $O(1)$ 的时间复杂度处理动态连通性问题。

竞赛中的应用场景

并查集是图论和数据结构中的“万金油”，常用于处理：

  • 动态连通性：随时询问图中两点是否连通，或者随时往图中加边。
  • 最小生成树 (Kruskal 算法)：按照边权排序，使用并查集判断加入某条边是否会形成环。
  • 求连通分量个数：遍历完所有边后，统计有多少人的“帮主”是自己。
  • 带权并查集 / 种类并查集：维护元素之间的相对关系（如：食物链中的捕食关系、敌人的敌人是朋友等）。
  • LCA 离线算法 (Tarjan)：利用并查集在 $O(N+Q)$ 时间内处理所有最近公共祖先查询。

模板代码（C++ 版本）

推荐使用路径压缩优化。注意：在调用 unite 前通常需要先用 find 检查是否已经在同一集合中。

#include 

using namespace std;

// 1. 定义全局数组。数组大小开到 N + 5 左右比较保险
const int MAXN = 100005;
int fa[MAXN];

// 2. 查找函数 (Find)：包含路径压缩
// 这一行逻辑很精简：如果我的父亲不是我，就递归找父亲，并顺便更新我的父亲
int find(int i) {
    return fa[i] == i ? i : fa[i] = find(fa[i]);
}

// 3. 合并函数 (Union)：合并两个元素所在的集合
void unite(int i, int j) {
    int root_i = find(i);
    int root_j = find(j);
    if (root_i != root_j) {
        fa[root_i] = root_j; // 让一个根节点指向另一个根节点
    }
}

int main() {
    int n = 10;

    // 4. 初始化：核心步骤，千万别忘！
    // iota 是  里的函数，可以直接把数组填成 0, 1, 2, 3...
    iota(fa, fa + n + 1, 0);

    // 使用示例
    unite(1, 2);
    unite(2, 3);

    if (find(1) == find(3)) {
        cout << "1 和 3 在同一个帮派" << endl;
    }

    return 0;
}

赛场避坑指南 (WA / TLE / MLE 警告)

1. 忘记初始化 (WA)

并查集最开始必须 for(i=1~n) fa[i] = i;。如果初始值全是 0，所有的 find 都会死循环或者指向节点 0。

2. 只有合并，没有路径压缩 (TLE)

如果没有 fa[x] = find(fa[x]) 这一步压缩，并查集可能会退化成一条很长的链，查询复杂度变成 $O(N)$，在大规模数据下必超时。

3. 合并了非根节点 (Logic Error)

在合并 $x$ 和 $y$ 时，应该先找到它们的根 $fx$ 和 $fy$，然后令 fa[fx] = fy。如果你直接令 fa[x] = y，会破坏原有的树形结构，导致逻辑错误。

4. 带权并查集的回溯污染 (WA)

在处理带权并查集（如食物链）时，路径压缩时权值的更新公式必须推导准确（通常是模运算），否则会导致关系错乱。

复杂度分析

  • 时间复杂度：
    • 同时使用路径压缩和按秩合并：单次操作近乎 $O(\alpha(N))$，其中 $\alpha$ 是反阿克曼函数，在实际应用中可以看作常数（小于 5）。
    • 仅使用路径压缩：最坏 $O(\log N)$。
  • 空间复杂度：$O(N)$，只需一个存储父节点的数组。
