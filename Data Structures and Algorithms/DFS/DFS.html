核心思想

生活类比：假设你在走一个巨大且没有地图的迷宫。你的策略是“不撞南墙不回头”：沿着一条路一直走到底，如果发现是死胡同，就退回到上一个岔路口（这个动作叫回溯），换另一条路继续走。直到找到出口或者遍历完所有的死胡同。

  • 广度优先 (BFS)：像水波纹一样，一层一层往外扩张（需要队列）。
  • 深度优先 (DFS)：像钻头一样，一条道走到黑，到底后原路拔出再钻下一个孔（需要栈/递归）。

算法本质：递归与回溯。隐式地利用了操作系统的函数调用栈来记录走过的“岔路口”，在穷举搜索和图的连通性分析中是绝对的基石。

竞赛中的应用场景

在算法竞赛中，DFS 主要解决“走迷宫”、“穷举所有可能”以及“树/图的遍历”问题。常见套路包括：

  • 暴力枚举 (搜索)：求全排列、求子集、凑数字组合等。当问题规模不大（通常 $N \le 20$）且需要列出所有合法方案时，DFS 是第一选择。
  • 图与网格遍历 (Flood Fill)：扫雷游戏中点开一片空白块、计算图像中的连通色块数量（连通分量）。
  • 树上操作：找树的重心、预处理 LCA（最近公共祖先）、树上动态规划（Tree DP）。由于树天然具有递归结构，DFS 是处理树结构最直观的工具。
  • 记忆化搜索：DFS 与动态规划的结合体。通过一个数组记录已经计算过的分支结果，如果下次再 DFS 到这里直接返回，从而将指数级复杂度降维到多项式级别。

必刷经典题单

以下题目涵盖了 DFS 在竞赛中的核心考点，特别是从纯暴力到带有“剪枝”优化的进阶：

  • Level 1: 纯享递归与回溯
    洛谷 P1706 - 全排列问题 DFS 模板
    评语：回溯法的启蒙题。深刻理解 used[i]=true（做选择）和 used[i]=false（撤销选择，恢复现场）的意义。
  • Level 2: 二维棋盘搜索
    洛谷 P1219 - 八皇后 经典回溯
    评语：世界级经典题。难点在于如何巧妙地用数组来 $O(1)$ 判定对角线上是否已经放置了皇后。
  • Level 3: 网格与连通块
    洛谷 P1331 - 海战 Flood Fill
    评语：典型的网格图 DFS，不仅要利用 DFS 找到连通的船只，还需要在遍历过程中判定形状是否合法。
  • Level 4: 剪枝的艺术
    洛谷 P1120 - 小木棍 高级剪枝
    评语：CSP/NOIP 必刷神题。不剪枝只能拿 20 分。需要极强的优化思维（可行性剪枝、最优性剪枝、排除等效冗余）来约束 DFS 庞大的搜索树。

模板代码（C++ 版本）

这是生成“全排列”的经典回溯模板。请死记 “递归终止 -> 遍历选项 -> 做出选择 -> 递归进入 -> 撤销选择” 这个标准五步框架。

#include <iostream>
#include <vector>

using namespace std;

int n = 3; 
vector<int> path; // 记录当前走过的路径 (相当于系统栈)
vector<bool> used(n + 1, false); // 记录数字是否已经被使用

void dfs(int depth) {
    // 1. 递归的终止条件 (Base Case: 走到底了)
    if (depth == n) {
        for (int x : path) cout << x << " ";
        cout << "\n";
        return;
    }

    // 2. 遍历当前步骤下的所有可能选项
    for (int i = 1; i <= n; i++) {
        if (!used[i]) { // 如果这个数字还没被用过
            
            // 3. 做出选择 (标记状态)
            used[i] = true;
            path.push_back(i);

            // 4. 进入下一层决策树
            dfs(depth + 1);

            // 5. 撤销选择 (回溯，恢复现场！这步是核心！)
            path.pop_back();
            used[i] = false;
        }
    }
}

int main() {
    dfs(0); // 从第 0 层深度开始搜索
    return 0;
}

赛场避坑指南 (WA / TLE / MLE 警告)

1. 忘记恢复现场 (WA)

如果不同分支共享同一个状态变量（如全局数组或引用传递），在子节点的 DFS 返回到父节点时，**必须**将修改过的状态改回来。否则上一个分支的污染会带入下一个分支，导致答案错误。

2. 缺失 Base Case / 忘标记 Visited (MLE / TLE)

在图的遍历中，图可能存在环。如果不使用 `visited` 数组记录走过的节点，DFS 会在环里无限循环，导致系统递归栈溢出（爆栈导致 Runtime Error / MLE）。另外，永远把 `if (终点)` 放在函数的开头。

3. 不懂剪枝硬搜 (TLE)

DFS 搜索树的规模往往是指数级 $O(2^N)$ 或阶乘级 $O(N!)$。当发现 $N > 20$ 且题目要求你用 DFS 时，一定要想办法剪枝。比如：当前花销已经大于已知最小答案了（最优性剪枝），或者当前状态无论如何也拼不凑出目标了（可行性剪枝），直接 `return` 结束该分支。

复杂度分析

  • 时间复杂度：
    • 遍历图与树：每个节点和每条边访问一次，时间 $O(V + E)$。
    • 组合穷举：视搜索树的大小而定，如全排列是 $O(N \times N!)$，子集生成是 $O(2^N)$。
  • 空间复杂度：由递归调用栈的最大深度决定。对于树/图，最坏情况下（退化成链）为 $O(V)$；对于排列组合通常为 $O(N)$。
