核心思想（一句废话没有）

生活类比：我们在玩“猜数字”游戏，范围是 1 到 100。你猜 50，我告诉你“太大了”；那你接下来肯定猜 25，绝不可能去猜 75。

  • 暴力法：从 1 猜到 100，最坏情况需要猜 $N$ 次，时间复杂度 $O(N)$。如果有 $M$ 次查询或 $N$ 极大，必吃 TLE。
  • 二分法：每次取中间值测试，将搜索空间直接砍半。最坏情况只需猜 $\log_2 N$ 次。

算法本质：利用单调性（或两段性）排除无效区间。将 $O(N)$ 的线性扫描优化为 $O(\log N)$ 的对数级探测。

开区间 l + 1 < r 终极模板

手写整数二分最容易陷入死循环！强烈推荐使用 (l, r) 开区间模板（又称红蓝染色法）。核心思想是：l 和 r 分别维护两种不同的颜色（合法与非法），每次将 mid 染成对应的颜色并移动边界。当 l + 1 == r 时，两者相遇，不留死角。

模型一：最小化答案 (找下界)

场景：找满足条件的最小值（例如：运输能力至少为多少才能运完）。
数组形态：0 0 0 0 1 1 1 1 (0代表不满足，1代表满足)。我们需要找第一个 1。
规则：l 维护 0，r 维护 1。最终答案在 r。

ll l = mn - 1; // 永远不满足的虚拟边界
ll r = mx + 1; // 永远满足的虚拟边界

while (l + 1 < r) {
    ll mid = (l + r) >> 1;
    if (check(mid)) {
        r = mid; // mid满足，说明答案<=mid
    } else {
        l = mid; // mid不满足，说明答案>mid
    }
}
// r 是第一个满足条件的最小值
cout << r << '\n';

场景：找满足条件的最大值（例如：木板最长能切多长）。

算法竞赛中，99% 的二分题都是“二分答案”。把求最优解问题转化为 check(mid) 的可行性判定问题。

  • Level 1: 基础测试 (STL 熟练度)
    洛谷 P2249 - 【深基13.例1】查找 模型一：找第一个大于等于
    评语：可以用来测试你的手写“最小化答案”模板，或者直接熟练使用 std::lower_bound。
  • Level 2: 最大化答案入门
    洛谷 P2440 - 木材加工 模型二：最大化长度
    评语：经典 11110000 形态。切的长度越短越能满足，越长越不能满足。套用模型二，求最大满足长度 l。
  • Level 3: 最小化最大值 (经典中的经典)
    AtCoder ABC 174 E - Logs 模型一：最小化答案
    评语：把原木切 $K$ 次，求最长木头的最小可能长度。限制越宽松（设定的长度越长），切的次数越少。典型的 00001111 形态，套用模型一，求最小合法长度 r。
  • Level 4: 最大化最小值 (NOIP名题)
    洛谷 P2678 - [NOIP2015 提高组] 跳石头 模型二：最大化最短距离
    评语：全国 Oier 的童年回忆。设定的最短跳跃距离 mid 越小越容易满足（移走石头少）。套用模型二，在 check(mid) 中判断移走石头数是否 $\le M$。
  • Level 5: 综合变形应用 (贪心+二分)
    Codeforces 1201C - Maximum Median 模型二：最大化中位数
    评语：CF 常见套路。二分最终的中位数 mid，然后计算把原数组中位数提升到 mid 需要多少代价，与允许的总代价 K 比较。

赛场避坑指南 (WA / TLE 警告)

1. l + r 溢出警告 (爆 int)

虽然 ll mid = (l + r) >> 1; 运行极快，但当 l 和 r 都接近 $2 \times 10^9$ 时，如果用 int 存储，l + r 会直接溢出成负数。在 XCPC 中，如果答案范围达到了 $10^9$，务必将 l、r、mid 都开成 long long。

2. 虚拟边界的取值

mn - 1 和 mx + 1 一定要能涵盖所有可能的情况！如果答案可能是 0，那 l 至少要从 -1 开始；如果答案最大可能到 $10^{18}$，r 就要开到 $10^{18} + 1$。

3. 单调性不严格

必须确保 check() 函数的结果呈现绝对的两段性（要么是 00001111，要么是 11110000）。如果你发现它不满足两段性（例如：1 0 1 1 0 0），说明不能直接二分，可能需要结合其他算法（如贪心、DP）来维持局部单调性，或者完全换思路。
