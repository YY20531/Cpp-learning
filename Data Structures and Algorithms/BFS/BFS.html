核心思想

生活类比：想象一颗石子投入平静的湖面，水波会一圈一圈向外扩散。或者想象你在社交网络上找人：先问你的直接好友（第一层），如果没找到，再问你好友的好友（第二层），以此类推。

  • 深度优先 (DFS)：不撞南墙不回头，用“栈”或“递归”实现。
  • 广度优先 (BFS)：齐头并进，稳扎稳打。它必须借助 队列 (Queue) 这种“先进先出 (FIFO)”的数据结构来实现。

算法本质：层序遍历与无权最短路。因为 BFS 是严格按照距离起点的“层级”由近及远向外扩展的，所以当你第一次碰到目标终点时，走过的步数一定是最少的。

竞赛中的应用场景

在算法竞赛中，只要看到题目中出现“最少步数”、“最短距离”、“层级”等字眼，且走一步的代价都相同（无权图），BFS 绝对是首选武器：

  • 网格迷宫寻路：在一个二维地图中，遇到墙壁不能走，求从起点到终点的最短步数。
  • 多源 BFS：地图上有多个起点同时开始向外扩散（例如：多处火源同时蔓延、腐烂的橘子传染周围的好橘子）。只需在一开始把所有起点都丢进队列即可。
  • 状态空间搜索 (隐式图)：比如经典的“八数码”问题、倒水问题。把棋盘的一个“局面”当成一个节点，每次移动得到的新局面当成邻居节点进行 BFS。
  • 拓扑排序：通过统计节点的入度，配合队列将入度为 0 的节点不断剥离，常用于解决任务调度、课程安排是否有环的问题。

必刷经典题单

以下题目涵盖了 BFS 从最基础的入队出队，到复杂状态判重的核心套路：

  • Level 1: 基础迷宫/层序遍历
    洛谷 B3625 - 迷宫寻路 BFS 入门
    评语：这道题是 BFS 的“Hello World”。它去掉了所有繁琐的干扰，让你集中精力练习如何正确地“入队、出队、标记访问”，并直观感受 BFS 像水波纹一样扩散的寻路逻辑。
  • Level 2: 二维迷宫最短路
    洛谷 P1443 - 马的遍历 方向数组 + 判重
    评语：棋盘 BFS 必刷题。掌握 dx[] 和 dy[] 方向数组的写法，以及越界判断。
  • Level 3: 多源并发 BFS
    洛谷 P1332 - 血色先锋队 多源 BFS
    评语：不要对每个起点分别做 BFS（会 TLE）。正确做法是初始时把所有感染源全部 `push` 进队列，然后自然蔓延。
  • Level 4: 状态压缩与哈希判重
    洛谷 P1379 - 八数码难题 状态图 BFS
    评语：进阶神题。将一个 $3 \times 3$ 的矩阵压缩成一个整数或字符串，利用 std::unordered_map 记录该状态的最少步数。

模板代码（C++ 版本）

这是二维网格迷宫寻找最短路的经典模板。核心口诀：“队首出队 -> 枚举邻居 -> 合法且未访问 -> 标记访问并入队”。

#include <iostream>
#include <queue>

using namespace std;

// 定义状态结构体
struct Node {
    int x, y;
    int step; // 记录从起点走到这里的步数
};

int main() {
    int n = 5, m = 5; // 5x5 的迷宫
    // 技巧：定义上下左右 4 个方向的偏移量
    int dx[4] = {-1, 1, 0, 0}; 
    int dy[4] = {0, 0, -1, 1};
    
    // 访问标记数组 (极度重要！)
    bool visited[10][10] = {false};
    
    queue<Node> q;
    
    // 1. 起点入队，并立即标记为已访问
    q.push({1, 1, 0});
    visited[1][1] = true;
    
    while (!q.empty()) {
        // 2. 取出队首元素
        Node curr = q.front();
        q.pop();
        
        // 如果到达终点，直接输出并结束
        if (curr.x == n && curr.y == m) {
            cout << "最短步数: " << curr.step << "\n";
            return 0;
        }
        
        // 3. 遍历四周相邻的节点
        for (int i = 0; i < 4; i++) {
            int nx = curr.x + dx[i];
            int ny = curr.y + dy[i];
            
            // 4. 越界检查 与 访问判断
            if (nx >= 1 && nx <= n && ny >= 1 && ny <= m && !visited[nx][ny]) {
                // (此处可增加遇到墙壁不走的判断)
                
                // 5. 标记为已访问，并推入队列
                visited[nx][ny] = true; 
                q.push({nx, ny, curr.step + 1});
            }
        }
    }
    
    cout << "无法到达终点\n";
    return 0;
}

赛场避坑指南 (MLE / TLE 警告)

1. 入队时不标记，出队时才标记 (超级大坑 - MLE/TLE)

新手最容易犯的错！必须在将节点推入队列 (push) 的同一瞬间，将其标记为 `visited = true`。如果等节点从队列里弹出来 (pop) 时才标记，同一个节点可能会被其他邻居重复 push 进队列无数次，导致队列瞬间爆炸，引发内存超限 (MLE) 或超时 (TLE)。

2. 滥用 BFS 处理带权图 (WA)

BFS 能求最短路的前提是“边权全部相等”（走一步代价都算 1）。如果走到 A 消耗 5 体力，走到 B 消耗 2 体力，BFS 就失效了。此时必须换用 Dijkstra 算法 (优先队列 BFS)。

3. 忽略边界与越界判断 (RE)

在使用 dx dy 扩展坐标时，永远记得先写 `nx >= 0 && nx < R` 的边界判断，再写 `!visited[nx][ny]` 的判断，否则会出现数组越界导致运行时错误 (Segment Fault)。

复杂度分析

  • 时间复杂度：$O(V + E)$。其中 $V$ 是节点数（网格的格子数），$E$ 是边数。每个节点最多入队出队一次，每条边最多被检查一次。
  • 空间复杂度：$O(V)$。队列中最坏情况下会装下整个图“最宽”的那一层的节点（例如完全二叉树的底层），外加一个大小为 $V$ 的 visited 访问数组。
